# Plain-English Chat CLI Plan

Goal: add a first chat-style interface so users can interact with the trading system in plain English instead of memorizing commands.

Architecture: implement a deterministic intent router first, not a freeform autonomous agent. The chat layer should parse common requests into existing pipeline/CLI actions and return friendly summaries. Keep the first version narrow and reliable.

Tech Stack: Python, Click, regex/string parsing, existing TradingPipeline API, lightweight file persistence for alpha notes.

Tasks:
1. Create a chat router module with intent parsing and execution helpers.
2. Support first intents: status, analyze symbol, options analysis, backtest, opportunity scan, save alpha note, help, exit.
3. Add a `chat` CLI command that runs a simple REPL.
4. Add tests for intent parsing and execution.
5. Verify compile, tests, and CLI help output.
